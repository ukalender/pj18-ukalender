Unsupervised Learning: Clustering STSL Football Players by Style¶

The project aims to profile the football players based on the Opta's player game statistics in order to figure out how many different player profiles exist beyond the traditional positions such as goalkeeper, defender, midfielder, and forward. I will focus on only forward players in the context of this study but other positions can be also analysed in the future to compare the model accuracy.

If you follow the football closely, you may have an idea about different style of forward players. Some of them act like a goal machine; they get smartly the position and have strong sense of opportunity to score while some forward players does not score much but contribute to goals scored by other team members. On the other hand there are some forward players who do not have abilities which are highly expected from forwards. At the end of the study, my aim is to make this profiling fed by Opta data which records all actions with ball in addition to traditional metrics such as goal and assists.

Machine learning algorithm like k-means clustering is relevant method to cluster such an unlabeled data.

Literature Review¶

There is limited number of similar study conducted on that topic but i managed to find two useful articles which will guide me during the project. In the first article [1], authors try to find relation between value of players and their actions in the game with ball. They criticize the traditional approaches which only consider popular metrics like goals and shots. Conclusion is that it is more suitable to value the player based on the actions that increase the team’s chance of scoring while it would be best to include also actions without ball like positioning.

Second article [2] focuses solely on performance of teams rather than players. They used sixteen different performance metrics from pass accuracy to yellow cards and classified the teams by using k-cluster analysis. Data contains only matches in the UEFA Champions League.

#Calculating Eigenvecors and eigenvalues of Covariance matrix
mean_vec = np.mean(pca_data, axis=0)
cov_mat = np.cov(pca_data.T)
eig_vals, eig_vecs = np.linalg.eig(cov_mat)

# Create a list of (eigenvalue, eigenvector) tuples
eig_pairs = [ (np.abs(eig_vals[i]),eig_vecs[:,i]) for i in range(len(eig_vals))]

# Sort from high to low
eig_pairs.sort(key = lambda x: x[0], reverse= True)

# Calculation of Explained Variance from the eigenvalues
tot = sum(eig_vals)
var_exp = [(i/tot)*100 for i in sorted(eig_vals, reverse=True)] # Individual explained variance
cum_var_exp = np.cumsum(var_exp) # Cumulative explained variance

Silhoutee Analysis¶

Silhouette analysis is a good tool to discover whether there is any trade-off between numbe rof clusters.

In [29]:

In [30]:
#Plot silhouette analysis for different number of clusters
from sklearn.metrics import silhouette_samples, silhouette_score

for i, k in enumerate([3,4,5]):
    fig, (ax1, ax2) = plt.subplots(1, 2)
    fig.set_size_inches(18, 7)
    
    # Run the Kmeans algorithm
    km = KMeans(n_clusters=k)
    labels = km.fit_predict(km_data)
    centroids = km.cluster_centers_

    # Get silhouette samples
    silhouette_vals = silhouette_samples(km_data, labels)

    # Silhouette plot
    y_ticks = []
    y_lower, y_upper = 0, 0
    for i, cluster in enumerate(np.unique(labels)):
        cluster_silhouette_vals = silhouette_vals[labels == cluster]
        cluster_silhouette_vals.sort()
        y_upper += len(cluster_silhouette_vals)
        ax1.barh(range(y_lower, y_upper), cluster_silhouette_vals, edgecolor='none', height=1)
        ax1.text(-0.03, (y_lower + y_upper) / 2, str(i + 1))
        y_lower += len(cluster_silhouette_vals)

    # Get the average silhouette score and plot it
    avg_score = np.mean(silhouette_vals)
    ax1.axvline(avg_score, linestyle='--', linewidth=2, color='green')
